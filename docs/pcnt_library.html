Overview

The Pulse Counter (PCNT) peripheral is designed to count the number of rising and/or falling edges of input signals without CPU intervention. The ESP32 has up to 8 PCNT units, each with 2 channels, allowing for versatile pulse counting applications.

Key Features

  • Hardware-based counting - No CPU intervention required
  • 16-bit signed counters (-32,768 to +32,767)
  • Up to 8 PCNT units (PCNT_UNIT_0 to PCNT_UNIT_7)
  • 2 channels per unit (PCNT_CHANNEL_0, PCNT_CHANNEL_1)
  • Configurable count modes (increment, decrement, hold)
  • Event-based interrupts (overflow, underflow, thresholds)
  • Input filtering for noise immunity
  • Maximum frequency: ~10-40MHz (depending on signal quality)

Core Functions

Configuration Functions

pcnt_unit_config()

esp_err_t pcnt_unit_config(const pcnt_config_t *pcnt_config);

Purpose: Configure a PCNT unit with specified parameters.

Parameters:
• pcnt_config: Pointer to configuration structure
Returns: ESP_OK on success, error code on failure
Note: This function will disable three events: PCNT_EVT_L_LIM, PCNT_EVT_H_LIM, PCNT_EVT_ZERO.

Complete Example: Frequency Counter with 32-bit Extension

This example demonstrates how to implement a frequency counter with 32-bit counting capability using manual overflow handling.

#include "driver/pcnt.h" #include "esp_intr_alloc.h" #include "soc/pcnt_struct.h" #define PCNT_INPUT_PIN 4 #define PCNT_HIGH_LIMIT 32767 static portMUX_TYPE pcnt_spinlock = portMUX_INITIALIZER_UNLOCKED; volatile long overflow_count = 0; // ISR for overflow handling static void IRAM_ATTR pcnt_isr_handler(void *arg) { portENTER_CRITICAL_ISR(&pcnt_spinlock); overflow_count++; PCNT.int_clr.val = BIT(PCNT_UNIT_0); portEXIT_CRITICAL_ISR(&pcnt_spinlock); } void setup() { Serial.begin(115200); // Configure PCNT pcnt_config_t pcnt_config = { .pulse_gpio_num = PCNT_INPUT_PIN, .ctrl_gpio_num = PCNT_PIN_NOT_USED, .lctrl_mode = PCNT_MODE_KEEP, .hctrl_mode = PCNT_MODE_KEEP, .pos_mode = PCNT_COUNT_INC, .neg_mode = PCNT_COUNT_DIS, .counter_h_lim = PCNT_HIGH_LIMIT, .counter_l_lim = -32768, .unit = PCNT_UNIT_0, .channel = PCNT_CHANNEL_0, }; // Initialize PCNT pcnt_unit_config(&pcnt_config); pcnt_event_enable(PCNT_UNIT_0, PCNT_EVT_H_LIM); pcnt_isr_register(pcnt_isr_handler, NULL, 0, NULL); pcnt_intr_enable(PCNT_UNIT_0); pcnt_counter_clear(PCNT_UNIT_0); pcnt_counter_resume(PCNT_UNIT_0); Serial.println("PCNT Frequency Counter initialized"); } // Get total count with overflow handling (32-bit) int32_t get_count_32bit() { portENTER_CRITICAL(&pcnt_spinlock); long overflow_copy = overflow_count; int16_t hardware_count = 0; pcnt_get_counter_value(PCNT_UNIT_0, &hardware_count); portEXIT_CRITICAL(&pcnt_spinlock); // Calculate total count int64_t total_64 = (overflow_copy * (int64_t)PCNT_HIGH_LIMIT) + hardware_count; // Convert to 32-bit signed with saturation if (total_64 > INT32_MAX) { return INT32_MAX; // +2,147,483,647 } else if (total_64 < INT32_MIN) { return INT32_MIN; // -2,147,483,648 } else { return (int32_t)total_64; } } // Get frequency (Hz) int32_t get_frequency() { static int32_t last_count = 0; int32_t current_count = get_count_32bit(); int32_t frequency = current_count - last_count; last_count = current_count; return frequency; } void loop() { int32_t freq = get_frequency(); Serial.printf("Frequency: %d Hz\n", freq); delay(1000); }

Best Practices

GPIO Selection

Category GPIO Pins Notes
✅ Recommended 4, 5, 16, 17, 18, 19, 21, 22, 23, 25, 26, 27, 32, 33 Internal pull-up/pull-down available
⚠️ Use with care 34, 35, 36, 39 Input-only, require external pull-ups
❌ Avoid 6-11 Connected to SPI flash
❌ Boot pins 0, 2, 12 Can affect boot sequence

High-Frequency Applications

  • Use external pull-up resistors (1kΩ-4.7kΩ) for 4MHz+ signals
  • Enable input filtering for noisy environments
  • Implement overflow handling for continuous counting
  • Use dedicated ISR for minimal latency
  • Keep ISR code minimal and use IRAM_ATTR

Error Handling

esp_err_t err = pcnt_unit_config(&config); if (err != ESP_OK) { Serial.printf("PCNT config failed: %s\n", esp_err_to_name(err)); }

Multi-Channel Applications

  • Each unit can have 2 channels
  • Use different units for independent counting
  • Share ISR between units for efficiency
  • Use unit-specific overflow variables

Limitations

  • 16-bit hardware counter: Requires software overflow handling for large counts
  • No built-in 32-bit accumulation: Manual implementation required
  • Shared interrupts: All events on a unit share one interrupt
  • Input frequency limit: ~10-40MHz depending on signal quality
  • Filter resolution: 10-bit (0-1023 APB clock cycles)
  • Maximum units: 8 units total (may be limited by specific ESP32 variant)

Conclusion

This documentation covers the essential PCNT functionality available in Arduino ESP32 Core 2.0.6. The PCNT peripheral provides powerful hardware-based pulse counting capabilities that are essential for high-frequency applications. For advanced applications requiring 32-bit counting or higher precision, implement software overflow handling as demonstrated in the examples.

Remember that while the hardware counter is limited to 16-bit, proper overflow handling can extend this to 32-bit or even 64-bit counting ranges, making it suitable for applications requiring long-term counting or high-frequency measurements.
