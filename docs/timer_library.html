ESP32 Hardware Timer API Documentation

Important: This documentation covers the ESP32 hardware timer driver API for PlatformIO with Arduino framework.

Clock Explanation:
  • CPU Clock: 240MHz (main processor frequency)
  • APB Clock: 80MHz (peripheral bus clock - used by timers)
  • Timer Base: APB_CLK_FREQ = 80MHz (what timers actually use)

Why 80MHz for timers? ESP32 uses different clock domains for power efficiency. Peripherals like timers, UART, SPI run on the APB (Advanced Peripheral Bus) clock at 80MHz, while the CPU cores run at 240MHz. This separation allows better power management and reduces electromagnetic interference.

Overview

The ESP32 has 2 timer groups with 2 timers each (total 4 hardware timers):

  • TIMER_GROUP_0: Contains TIMER_0 and TIMER_1
  • TIMER_GROUP_1: Contains TIMER_0 and TIMER_1

Why 80MHz Timer Clock vs 240MHz CPU Clock?

ESP32 Clock Architecture:
┌─────────────────┐ ┌──────────────────┐ │ CPU Cores │ │ Peripherals │ │ (240 MHz) │ │ (80 MHz) │ │ │ │ │ │ • Core 0 │ │ • Timers │ │ • Core 1 │ │ • UART │ │ • Cache │ │ • SPI │ │ • Memory │ │ • I2C │ │ │ │ • GPIO │ └─────────────────┘ └──────────────────┘ │ │ └───────┬───────────────┘ │ ┌───────────▼────────────┐ │ Clock Generator │ │ (PLL + Dividers) │ │ │ │ XTAL: 40MHz │ │ CPU_CLK: 80/160/240MHz │ │ APB_CLK: 80MHz (fixed) │ │ REF_CLK: 1MHz │ └────────────────────────┘
Technical Reasons for 80MHz Peripheral Clock:
  • Power Efficiency: Peripherals don't need 240MHz - wastes power
  • EMI Reduction: Lower frequencies reduce electromagnetic interference
  • Timing Stability: Independent of CPU frequency scaling
  • Compatibility: Standard APB bus frequency across ESP32 family
  • Precision: 80MHz provides good balance of resolution vs range

Key Concepts:
  • CPU Clock: 240 MHz (configurable: 80, 160, 240 MHz)
  • APB Clock: 80 MHz (peripheral clock - fixed for most configurations)
  • Timer Base Clock: APB_CLK_FREQ = 80 MHz
  • Counter Width: 64-bit (0 to 18,446,744,073,709,551,615)
  • Prescaler Range: 2 to 65,536
  • Resolution Formula: Resolution = (1 / 80MHz) × Prescaler
📚 Clock Domain Explanation:

ESP32 uses multiple clock domains for optimal performance:

  • CPU_CLK (240MHz): Main processor cores, cache, memory
  • APB_CLK (80MHz): Timers, UART, I2C, SPI, GPIO
  • REF_CLK (1MHz): RTC, deep sleep timers
  • XTAL_CLK (40MHz): Crystal oscillator reference

This design allows peripherals to run independently of CPU frequency changes, ensuring consistent timing even when CPU enters power-saving modes.

Basic Timer Operations

Initialization & Configuration

esp_err_t timer_init(timer_group_t group_num, timer_idx_t timer_num, const timer_config_t *config)

Initializes and configures a hardware timer.

Parameters:
  • group_num: Timer group (TIMER_GROUP_0 or TIMER_GROUP_1)
  • timer_num: Timer index (TIMER_0 or TIMER_1)
  • config: Pointer to timer configuration structure
Returns: ESP_OK on success, ESP_ERR_INVALID_ARG on parameter error
Example:
timer_config_t timer_config = { .divider = 80, // 80MHz / 80 = 1MHz (1µs resolution) .counter_dir = TIMER_COUNT_UP, // Count upward .counter_en = TIMER_PAUSE, // Start paused .alarm_en = TIMER_ALARM_DIS, // Disable alarm .auto_reload = TIMER_AUTORELOAD_DIS // No auto-reload }; timer_init(TIMER_GROUP_0, TIMER_0, &timer_config);
esp_err_t timer_set_divider(timer_group_t group_num, timer_idx_t timer_num, uint32_t divider)

Sets the clock divider (prescaler) for the timer.

Parameters:
  • divider: Clock divider value (2 to 65,536)
Common Prescaler Values:
  • 80 → 1µs resolution (1 MHz)
  • 800 → 10µs resolution (100 kHz)
  • 8000 → 100µs resolution (10 kHz)
  • 80000 → 1ms resolution (1 kHz)

Timer Control

esp_err_t timer_start(timer_group_t group_num, timer_idx_t timer_num)

Starts the timer counter.

Usage:
timer_start(TIMER_GROUP_0, TIMER_0); // Start counting
esp_err_t timer_pause(timer_group_t group_num, timer_idx_t timer_num)

Pauses the timer counter (preserves current value).

Usage:
timer_pause(TIMER_GROUP_0, TIMER_0); // Pause counting
esp_err_t timer_set_counter_value(timer_group_t group_num, timer_idx_t timer_num, uint64_t load_val)

Sets the timer counter to a specific value.

Parameters:
  • load_val: Value to set the counter to (0 to 2^64-1)
Usage for Stopwatch Reset:
timer_set_counter_value(TIMER_GROUP_0, TIMER_0, 0); // Reset to zero

Reading Timer Values

esp_err_t timer_get_counter_value(timer_group_t group_num, timer_idx_t timer_num, uint64_t *timer_val)

Reads the current counter value in timer ticks.

Parameters:
  • timer_val: Pointer to store the counter value
Usage:
uint64_t timer_val; timer_get_counter_value(TIMER_GROUP_0, TIMER_0, &timer_val); uint64_t microseconds = timer_val; // If prescaler = 80
esp_err_t timer_get_counter_time_sec(timer_group_t group_num, timer_idx_t timer_num, double *time)

Reads the current counter value converted to seconds.

Parameters:
  • time: Pointer to store time in seconds (double precision)
Usage:
double elapsed_seconds; timer_get_counter_time_sec(TIMER_GROUP_0, TIMER_0, &elapsed_seconds);

Advanced Features

Alarm Functions

esp_err_t timer_set_alarm_value(timer_group_t group_num, timer_idx_t timer_num, uint64_t alarm_value)

Sets an alarm value that triggers an interrupt when reached.

Usage for 1-second alarm:
// With prescaler = 80 (1µs resolution) timer_set_alarm_value(TIMER_GROUP_0, TIMER_0, 1000000); // 1 second timer_set_alarm(TIMER_GROUP_0, TIMER_0, TIMER_ALARM_EN);

Interrupt Handling

esp_err_t timer_isr_callback_add(timer_group_t group_num, timer_idx_t timer_num, timer_isr_t isr_handler, void *arg, int intr_alloc_flags)

Adds an ISR callback for timer interrupts (recommended method).

Parameters:
  • isr_handler: Callback function (returns bool for task yield)
  • arg: Parameter passed to callback
  • intr_alloc_flags: Interrupt allocation flags
Example Callback:
bool IRAM_ATTR timer_callback(void *args) { // Handle timer interrupt // Return true if higher priority task should run return false; } timer_isr_callback_add(TIMER_GROUP_0, TIMER_0, timer_callback, NULL, 0);

Complete Usage Examples

Simple Stopwatch Implementation

#include "driver/timer.h" class HardwareStopwatch { private: timer_group_t group; timer_idx_t timer; bool running; public: void init() { group = TIMER_GROUP_0; timer = TIMER_0; running = false; timer_config_t config = { .divider = 80, // 1µs resolution .counter_dir = TIMER_COUNT_UP, .counter_en = TIMER_PAUSE, .alarm_en = TIMER_ALARM_DIS, .auto_reload = TIMER_AUTORELOAD_DIS }; timer_init(group, timer, &config); timer_set_counter_value(group, timer, 0); } void start() { if (!running) { timer_start(group, timer); running = true; } } void stop() { if (running) { timer_pause(group, timer); running = false; } } void reset() { timer_set_counter_value(group, timer, 0); } uint64_t getMicroseconds() { uint64_t timer_val; timer_get_counter_value(group, timer, &timer_val); return timer_val; // Direct microseconds with prescaler=80 } double getSeconds() { double time_sec; timer_get_counter_time_sec(group, timer, &time_sec); return time_sec; } String getFormattedTime() { uint64_t us = getMicroseconds(); uint32_t hours = us / 3600000000ULL; uint32_t minutes = (us % 3600000000ULL) / 60000000ULL; uint32_t seconds = (us % 60000000ULL) / 1000000ULL; uint32_t milliseconds = (us % 1000000ULL) / 1000ULL; return String(hours) + ":" + String(minutes) + ":" + String(seconds) + "." + String(milliseconds); } }; // Usage example: HardwareStopwatch stopwatch; void setup() { Serial.begin(115200); stopwatch.init(); } void loop() { // Start stopwatch on button press or proximity detection stopwatch.start(); delay(1000); // Simulate some operation // Display elapsed time Serial.println("Elapsed: " + stopwatch.getFormattedTime()); // Stop and reset for next cycle stopwatch.stop(); stopwatch.reset(); delay(2000); }
Important Notes:
  • Timer interrupts should use IRAM_ATTR for ISR functions
  • Timer Groups 0 and 1 may be used by WiFi/Bluetooth - check compatibility
  • Use timer_deinit() to clean up when done
  • 64-bit counter provides ~584,942 years at 1µs resolution
