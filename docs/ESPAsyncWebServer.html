ESPAsyncWebServer.h – API Overview

Version: 3.0.6 (Defines: ASYNCWEBSERVER_VERSION_MAJOR=3 / MINOR=0 / REVISION=6)

This header defines the asynchronous HTTP/WebSocket/EventSource server classes for ESP32 / ESP8266 / RP2040 within the Arduino environment.

Integration Notes

  • Include header: #include <ESPAsyncWebServer.h>
  • Requires AsyncTCP (ESP32) or ESPAsyncTCP (ESP8266).
  • Use AsyncWebServer server(80); server.begin(); to start.
  • Route example: server.on("/ping", HTTP_GET, [](auto* req){ req->send(200,"text/plain","pong"); });
  • Static files: server.serveStatic("/static", LittleFS, "/www");
  • WebSocket/EventSource provided via included headers (AsyncWebSocket.h / AsyncEventSource.h).

Minimal Usage Example

// Basic setup
#include <ESPAsyncWebServer.h>

AsyncWebServer server(80);

void setup() {
  Serial.begin(115200);
  WiFi.begin("ssid","pass");
  while(WiFi.status()!=WL_CONNECTED) delay(200);

  server.on("/", HTTP_GET, [](AsyncWebServerRequest* req){
    req->send(200, "text/plain", "Hello Async");
  });

  server.on("/json", HTTP_GET, [](AsyncWebServerRequest* req){
    req->send(200, "application/json", "{\"ok\":true}");
  });

  server.begin();
}

void loop() {}

Performance Considerations

  • Non-blocking design: callbacks fire inside AsyncClient events.
  • Large responses should use stream or chunked APIs to avoid RAM exhaustion.
  • Use template processor for lightweight server-side substitution without full parsing costs.

Security Considerations

  • Supports basic and digest authentication per handler.
  • Always validate upload sizes to prevent memory issues.
  • Add HTTPS/TLS via AsyncTCP SSL if enabled (check ASYNC_TCP_SSL_ENABLED).
